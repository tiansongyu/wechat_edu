# 家教直聘小程序使用说明

这是一款帮助家长寻找老师、帮助老师寻找家教需求的小程序。家长和老师使用同一个微信账号登录，可以随时切换身份，资料和业务记录会保存在平台数据库中。

文中的“实际示例”来自当前保留的数据库样例或已经跑通的 Docker 集成测试，不是只用于展示的前端假数据。临时测试账号在验证后已清理，但同一测试可以重复执行。

## 先认识五个入口

| 入口 | 可以做什么 | 实际示例与结果 |
| --- | --- | --- |
| 首页 | 搜索和筛选家教信息，查看详情、收藏或分享 | 输入“函数”，同时选择数学/物理、南山区和课结，只返回符合条件的数学需求，英语对照需求被排除 |
| 地图寻单 | 根据当前位置查看附近的家教信息 | 数据库样例保存了南山区科技园的经纬度；附近接口能返回距离，匹配前只显示模糊位置 |
| 发布 | 家长发布家教需求，老师发布个人求带信息 | 当前样例中“高一数学辅导（双休 / 可长期）”已通过审核并正常展示 |
| 消息 | 查看平台通知和双方沟通会话 | 老师提交申请后自动生成会话，家长回复“请先说明函数模块的学习计划”并成功从数据库回读 |
| 我的 | 修改头像昵称、切换身份、完善资料，查看报名、收藏、发布、预约和评价 | 开发者工具中可看到资料、身份和业务数量；头像、昵称、角色切换及四类业务列表均通过测试 |

## 家长怎么使用

### 1. 完善家长和学生资料

进入“我的”，确认当前显示“家长版”，点击“身份与匹配资料”。可以填写：

- 学生年级、学校和目前学习情况；
- 薄弱科目、学习目标和期望成绩；
- 学习习惯、性格特点和方便上课的时间；
- 对老师的要求；
- 常用省、市、区和详细地点。

资料越完整，老师越容易判断自己是否适合。

**实际示例：** 当前数据库中的家长“林女士”为学生“小林”填写了高一、数学函数题容易丢分、目标进入班级前二十、周六下午方便上课等资料。数据保存后重新读取内容一致，说明学生资料已经与数据库打通。

### 2. 发布家教需求

点击底部“发布”，填写科目、年级、课酬、结算方式、上课时间、学生情况和授课地点，然后提交平台审核。

- 审核通过后，信息才会公开展示；
- 审核未通过时，可以根据审核意见修改后重新提交；
- 已发布的信息需要修改时，点击“申请修改”；审核期间旧内容继续展示，通过后才显示新内容；
- 不再需要老师时，可以关闭发布。

**实际示例：** 林女士发布了“高一数学辅导（双休 / 可长期）”，区域为深圳市南山区科技园，课酬 200 元/小时、周结，重点辅导几何和函数。该记录目前以“已发布”状态保留在数据库并能在首页读取。

**修改示例：** 集成测试把一个已发布需求申请改为“审核后新标题”。审核前老师仍读取到旧标题，后台通过后才读取到新标题，证明修改不会绕过审核直接覆盖公开内容。

### 3. 查看和处理老师申请

进入“发布”找到对应需求，点击“报名管理”：

- 查看老师头像、资料、认证状态和申请说明；
- 先通过“平台沟通”询问时间、教学方式等信息；
- 确认合适后接受申请，不合适可以拒绝；
- 处理结果会同步通知老师。

**实际示例：** 陈老师对高一数学需求提交申请后，家长端报名管理能看到老师资料和申请记录。另一套完整集成场景中，家长先沟通再点击接受，申请状态由“待处理”变为“已接受”，同时自动创建预约。

### 4. 确认合作和评价

接受老师申请后，双方可以继续沟通并建立预约。预约完成后，家长和老师都需要点击“确认完成”。双方确认且没有争议后，才可以提交真实合作评价。

**实际示例：** 家长和老师依次确认预约、分别确认完成后，预约状态成功变为“已完成”。家长随后提交 5 星评价：“函数讲解清晰，课后反馈完整，合作体验很好。”评价可以在老师公开口碑中正确看到。

## 教师怎么使用

### 1. 完善教师资料并提交认证

进入“我的”，切换到“老师版”，点击“身份与匹配资料”。可以填写：

- 姓名、学校、专业、学历和教学年限；
- 擅长科目、课时价格和个人介绍；
- 教学风格、教学成果和考试辅导经验；
- 可用时间、授课方式、课程形式和语言能力；
- 使用省、市、区滚动控件添加一个或多个服务区域；
- 上传学历、教师资格等认证材料。

保存后资料会提交平台审核。教师认证通过后，才能正式申请家长发布的需求。

**实际示例：** 当前数据库中的“陈老师”填写了华南师范大学、数学教育、本科、4 年教龄、擅长数学、上门/在线授课和南山区服务范围，认证状态为“已通过”。集成测试还成功保存并回读了南山区、福田区两个服务区域。

### 2. 搜索适合的家教需求

首页搜索框支持输入科目、年级、区域、学生情况或发布人名称。还可以：

- 同时选择多个科目；
- 按区域、年级和结算方式筛选；
- 按最新发布或课酬高低排序；
- 使用地图查看附近需求；
- 收藏或分享感兴趣的信息。

如果筛选后没有结果，可以点击“重置筛选”重新查看。

**实际示例：** 测试同时创建数学和英语两条需求，然后搜索“函数”，多选数学/物理，选择南山区、课结并按价格排序。结果只包含数学需求，证明关键词和多个筛选条件可以一起工作。

### 3. 提交申请并沟通

打开需求详情，点击“立即申请”，填写简短的自我介绍、教学经验和可上课时间。

- 提交申请后会自动创建平台会话；
- 申请说明会成为会话中的第一条消息；
- 家长处理前可以继续沟通，也可以取消本次申请；
- 家长接受后，可以继续确认具体课程安排；
- 被拒绝或已取消的申请会保留状态记录。

**实际示例：** 老师填写“我有六年数理教学经验，周六下午可先沟通学习目标。”并提交申请。系统自动创建会话，这段申请说明成为第一条消息；家长在接受前即可回复，双方刷新后都能从数据库读取消息。

### 4. 发布老师求带信息

老师也可以在“发布”中发布自己的求带信息，介绍擅长科目、服务区域、课酬和可上课时间。信息同样需要平台审核，通过后家长可以查看老师详情和真实合作评价。

**实际示例：** 老师可以填写“初高中数学提分与习惯培养”、南山区、200 元/小时、周末下午和一对一授课。提交后先显示“待审核”，后台审核通过后才会作为老师求带信息展示；该发布及审核接口已包含在完整工作流测试中。

### 5. 完成合作和评价

老师可以在“我的—预约”中确认预约、确认完成、取消预约或发起争议。双方都确认完成且没有争议后，可以对家长提交合作评价。

**实际示例：** 老师在已完成预约中提交 5 星评价：“家长目标明确，沟通及时，课程安排合理。”家长的“我收到的评价”成功显示该评价；同一预约重复提交会被幂等保护，不会生成两条评价。

## 一次完整合作是怎样的

1. 家长发布需求并通过平台审核。
2. 老师找到需求并提交申请。
3. 家长查看申请，双方在平台内沟通。
4. 家长接受合适的老师，双方确认预约。
5. 课程完成后，家长和老师分别确认完成。
6. 双方提交真实合作评价。

**完整实测结果：** `提交申请 → 自动建会话 → 家长回复 → 接受申请 → 创建预约 → 老师确认 → 家长确认完成 → 老师确认完成 → 双方评价` 已在一次集成测试中连续执行成功，最终预约状态为“已完成”，双方 5 星评价均正确显示。

## 常见状态说明

### 发布状态

- 待审核：平台正在检查发布内容。
- 已发布：其他用户可以正常看到。
- 需修改：请根据审核意见完善后重新提交。
- 已关闭：信息不再公开，可以按需要重新提交。

**状态示例：** 新发布时为“待审核”；后台通过后变为“已发布”；拒绝时显示审核意见并进入“需修改”；用户主动关闭后不再出现在公开列表。

### 申请状态

- 待处理：等待家长接受或拒绝。
- 已接受：可以进入下一步沟通和预约。
- 未选中：家长本次没有接受申请。
- 已取消：老师主动取消了申请。

**状态示例：** 陈老师的保留样例为“已接受”；自动化测试另外验证了待处理申请可接受、拒绝或由老师取消，并且重复点击不会重复改变数据库状态。

### 预约状态

- 待确认：等待双方确认安排。
- 已确认：预约已经成立。
- 已完成：双方已经确认课程完成。
- 已取消：预约已经取消。
- 有争议：平台会保留业务记录，争议解决前不能正常评价。

**状态示例：** 当前保留预约为“已完成”。评价专项测试还把预约设置为“有争议”，系统正确阻止正常评价并从公开评价统计中排除该预约。

## 账号、头像和隐私

- 首次进入使用微信登录；头像可以选择微信头像或重新选择图片，昵称可以随时修改；
- 同一个账号可以切换家长版和老师版，切换身份不会自动执行申请、接受等业务操作；
- 匹配成功前，平台默认隐藏详细地址并降低地图位置精度；
- 联系方式默认隐藏，建议优先使用平台消息沟通；
- 评价只来自平台内已经完成的真实预约，公开评价不展示评价人的昵称和头像；
- 如果评价包含虚假、广告、骚扰或隐私内容，可以提交举报等待平台处理。

## 常见问题

**为什么老师不能申请？** 先确认当前为老师版、教师资料已经审核通过，并且需求仍处于已发布状态。

**为什么看不到新的发布内容？** 新发布和已发布内容的修改都需要平台审核，请查看发布状态和审核意见。

**为什么不能评价？** 家长和老师都要确认课程完成，并且预约不能处于争议状态。

**搜索不到内容怎么办？** 清空关键词或点击“重置筛选”，再逐个添加科目、区域等条件。

**可以同时当家长和老师吗？** 可以。在“我的”页面切换身份即可，两种身份的数据会分别展示。
